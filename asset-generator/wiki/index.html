üöÄ Quick Start

Using the parameter system in 3 steps:

// 1. Generate deterministic seed
const seedSystem = new SeedSystem();
const seed = seedSystem.generateSeed('asteroid', 'ORE_T1_001', 'iron', 0);

// 2. Load parameters and apply variance
const manager = new ParameterManager(asteroidParams);
const rng = seedSystem.createRNG(seed);
const params = manager.getAllRandomized(rng);

// 3. Generate asset
generateAsteroid(params);

üîë Seed System

The seed system ensures 100% reproducibility. Same seed always produces the same output.

CRC32-based Seed Generation

Seeds are generated by hashing asset identifiers using CRC32. This produces a 32-bit deterministic seed. 

// Generate seed from identifiers
const seed = seedSystem.generateSeed(
    'asteroid',      // Asset type
    'ORE_T1_001',   // Item ID
    'iron',         // Variant
    0               // Frame number (0-7 for animations)
);
// Result: 2847563891 (always the same for these inputs)
Seeded Random Number Generator (LCG)

Linear Congruential Generator with parameters from Numerical Recipes. Generates uniform, gaussian, and other distributions. 

const rng = seedSystem.createRNG(seed);

// Generate different types of random values
const float = rng.nextFloat(0, 1);        // [0, 1)
const int = rng.nextInt(1, 10);          // [1, 10]
const bool = rng.nextBool(0.7);          // 70% true
const choice = rng.choice(['a', 'b']);    // random element
const gaussian = rng.nextGaussian(0, 1);  // normal distribution

üìä Variance System

Each parameter has variance_min and variance_max values that define natural variation bounds.

Example: Asteroid Elongation
{
  "elongation_x": {
    "type": "float",
    "min": 0.5,
    "max": 2.0,
    "default": 1.0,
    "variance_min": -0.2,  // Can go down to 0.8
    "variance_max": 0.2    // Can go up to 1.2
  }
}

When using manager.getRandomizedValue('elongation_x', rng), the system will:

  1. Take the base value: 1.0
  2. Generate random variance: rng.nextFloat(-0.2, 0.2)
  3. Add variance to base: 1.0 + variance
  4. Clamp to parameter bounds: min(max(result, 0.5), 2.0)

ü™® Asteroid Parameters (27 Total)

Procedural asteroid shapes with full control over geometry, surface, and materials.

Base Shape
8 Parameters ‚ñº
shape_type
enum 7 values

Base geometric primitive for asteroid generation. Determines the fundamental shape before deformations.

Values: sphere, ellipsoid, oblong, irregular, fragmented, crystalline, potato
Default: irregular
elongation_x
float 0.5 - 2.0

Stretching along X-axis. Values < 1.0 compress, > 1.0 elongate.

Default: 1.0
Variance: -0.2 to +0.2
elongation_y
float 0.5 - 2.0

Stretching along Y-axis. Creates tall or flat asteroids.

Default: 1.0
Variance: -0.2 to +0.2
asymmetry
float 0.0 - 1.0

Overall shape irregularity. 0 = perfectly symmetric, 1 = maximum asymmetry.

Default: 0.3
Variance: -0.1 to +0.3
roundness
float 0.0 - 1.0

Edge sharpness control. 0 = sharp angular edges, 1 = smooth rounded edges.

Default: 0.5
Visual Effect: Controls SDF smoothing
facet_count
int 0 - 32

Polygonalization for crystalline appearance. 0 = smooth surface, higher values = more facets.

Default: 0
Use Case: High-purity crystal asteroids
twist_angle
float -180 to +180

Rotation/torsion along Y-axis in degrees. Creates twisted, tornado-like shapes.

Default: 0
Variance: -30 to +30
